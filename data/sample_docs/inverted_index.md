# 倒排索引

倒排索引是信息检索系统中最常用的索引结构之一。它不再以文档为中心保存全部文本，而是以词项为入口，记录每个词项出现在哪些文档或文本片段中。这样，当用户输入查询词时，系统可以直接访问对应的文档列表，而不必逐篇扫描所有资料。

倒排表通常由词典和文档列表组成。词典保存系统识别出的词项，例如“索引”“检索”“文档列表”等；文档列表则保存包含该词项的文档编号、出现位置或词频等信息。对于课程资料检索来说，倒排索引可以帮助学生快速定位包含关键概念的段落。

查询效率是倒排索引的重要优势。如果没有索引，系统需要对所有文本片段逐一匹配关键词；有了倒排索引后，检索过程可以转化为多个文档列表的合并、求交或排序。布尔检索、短语检索和相关性排序都可以建立在倒排索引之上。

在实际系统中，倒排索引还会结合分词、停用词过滤和词项归一化等预处理步骤。中文资料需要先进行中文分词，才能把连续文本转换为可用于索引和查询匹配的词项。
